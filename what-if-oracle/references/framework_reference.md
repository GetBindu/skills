# What-If Oracle Framework Reference

## 0·IF·1 Framework

- **0**: The unexpressed potential state
- **1**: The expressed current reality
- **IF**: The conditional transformation between them

## Scenario Branch Types

| Symbol | Branch | Purpose |
|--------|--------|---------|
| Ω | Best Case | Maximum upside |
| α | Likely Case | Most probable path |
| Δ | Worst Case | Maximum downside |
| Ψ | Wild Card | Black swan territory |
| Φ | Contrarian | Opposite of consensus |
| ∞ | Second Order | Cascading ripple effects |

## Analysis Modes

| Mode | Branches | Use Case |
|------|----------|----------|
| Quick | Ω α Δ | Fast 3-branch analysis |
| Deep | All 6 | Comprehensive scenario planning |
| Reverse | α Ω Φ | Backward mapping from desired outcomes |
| Competitive | Multi-stakeholder | Multiple competing perspectives |

## Golden Ratio Principle

Allocate 61.8% attention to the primary scenario, 38.2% to alternatives.

## Per-Branch Analysis Fields

For each scenario branch, provide:
1. **Probability** (0-100%)
2. **Narrative** (what happens)
3. **Key assumptions** (what must be true)
4. **Trigger conditions** (observable signals)
5. **Consequences timeline** (immediate / short-term / long-term)
6. **Required responses** (actions to take)
7. **Overlooked insight** (the 1% most miss)
